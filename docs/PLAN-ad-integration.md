# 📋 [PLAN] 광고 시스템 고도화 및 배치 (Ad Integration)

이 문서는 "이 주사 ∙ 어디가 싸요?" 앱의 광고 수익화를 위한 전면, 배너(이미지/문구) 광고의 기술적 구현 및 배치 전략을 정의합니다.

---

## 🚀 개요
- **목표**: 토스 인앱 광고 2.0 정책을 준수하며 사용자 경험을 최적화한 광고 배치.
- **제외 사항**: 보상형 광고(Rewarded Ad)는 일체 사용하지 않으며 관련 로직 파기.
- **핵심 정책**: 
  - 지역 변경 시 전면 광고 노출.
  - 리스트 내 이미지 배너 주기적 노출 (3개당 1개, 최대 2개).
  - 상세 정보 조회 빈도 기반 전면 광고 노출 (5회당 1회).

---

## 🛠 Phase 1: 기반 구조 설계 (Architecture)
1.  **디렉토리 재구성**:
    - `src/features/reward-ad` 삭제.
    - `src/features/ads` 생성: 광고 관련 컴포넌트, 훅, 상수 관리.
2.  **광고 설정 관리**:
    - `src/features/ads/adConfig.ts`: 광고 유닛 ID(테스트 및 상용) 및 노출 정책 변수 정의.
3.  **공통 컴포넌트 개발**:
    - `TossBannerAd.tsx`: `variant`('card' vs 'expanded') 및 `type`('image' vs 'text') 지원.
4.  **광고 제어 훅 개발**:
    - `useTossAds.ts`: 전면 광고(Interstitial) 로드 및 노출 제어.
    - `useAdFrequency.ts`: 상세 정보 조회 횟수 카운팅 로직.

## 🛠 Phase 2: 노출 로직 구현 (Placement)
1.  **전면 광고 (Interstitial)**:
    - `features/region`에서 지역 선택 후 검색 실행 시점에 호출.
    - 상세 정보 5회 조회 시점(Counter % 5 === 0)에 호출.
2.  **이미지 강조형 배너 (Native Card)**:
    - `HospitalList` 렌더링 시 인덱스 기반으로 배너 삽입.
    - `index % 3 === 2` 조건을 만족할 때 배너 컴포넌트 삽입.
    - 최대 2개 노출을 위한 카운트 가드 적용.
3.  **문구 강조형 배너 (Fixed Bottom)**:
    - 메인 레이아웃 하단에 `variant="expanded"` 스타일로 고정.

## 🛠 Phase 3: 테스트 및 안정화 (QA)
1.  **Test ID 검증**: 모든 광고가 `ait-ad-test-`로 시작하는 ID로 정상 동작하는지 확인.
2.  **Fallback 구현**: 토스 앱 외부 환경에서 광고 영역이 보이지 않거나 오류가 발생하지 않도록 `.isSupported()` 체크.
3.  **Performance**: 광고 로딩이 리스트 스크롤이나 화면 전환 속도에 영향을 주지 않는지 확인.
